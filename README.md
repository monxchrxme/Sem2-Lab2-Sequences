# Полиморфные абстрактные типы данных (Лабораторная работа №2 - Sequences)

Система линейных структур данных на C++, предоставляющая унифицированный интерфейс `Sequence<T>` для работы с динамическими массивами и связными списками, с поддержкой парадигм Mutable и Immutable объектов

## Сборка и запуск

Проект использует систему сборки CMake. Все команды выполняются из корневой директории проекта.

**1. Конфигурация и сборка проекта**
```bash
cmake -B build             # Конфигурация (выполняется единожды или при изменении CMakeLists.txt)
cmake --build build        # Компиляция проекта
```
2. Запустить только исполняемую программу (интерфейс):
```
./build/src/Lab2_app       # Запуск основного исполняемого файла
```

3. Запуск и фильтрация тестов
```
cd build && ctest --output-on-failure                         # Запуск всех тестов с выводом ошибок
./build/tests/run_tests --gtest_filter="ArraySequenceTest*"   # Запуск конкретного набора тестов (Google Test)
```
## Архитектура и детали реализации
1. Строгая семантика мутабельности

Реализовано строгое разделение на изменяемые (Mutable) и неизменяемые (Immutable) последовательности. Операции, возвращающие новую последовательность (get_subsequence, concat), наследуют парадигму базового объекта (*this):

get_subsequence():
- Immutable -> get_subsequence() возвращает Immutable
- Mutable -> get_subsequence() возвращает Mutable

concat() (тип результата определяется левым операндом):
- Mutable -> concat(Any) возвращает Mutable (расширение существующего объекта)
- Immutable -> concat(Any) возвращает Immutable (создание нового объекта)

2. Специализация BitSequence

BitSequence — конкретная реализация Sequence (Sequence<Bit>) с дополнительными битовыми операциями (bitwise_and, bitwise_or, bitwise_xor, bitwise_not)

- Реализован как наследник MutableArraySequence<Bit>, однако в логике UI и трансформаций обрабатывается как immutable (все методы возвращают новый объект)
- Поддерживает инициализацию через битовые строки (например, "10110")

3. Управление памятью в ArraySequence

Низкоуровневая работа с памятью для ArraySequence полностью инкапсулирована внутри класса DynamicArray, который оперирует понятиями size (фактическое число элементов) и capacity (размер выделенной памяти)

Стратегия реаллокации (resize):

- Если new_size <= capacity, изменяется только логический size, память не перевыделяется
- Если new_size > capacity, выделяется новый блок памяти размером max(new_size, capacity * 2). Старые данные перемещаются, старый блок удаляется

4. Итераторы (IEnumerator<T>)

Итераторы реализованы через паттерн Nested Classes и предоставляют доступ только для чтения (отсутствует set_current()), что гарантирует безопасность обхода Immutable-структур
- ArraySequence: Реализован поверх самого интерфейса массива (использует индексный доступ, сложность O(1))
- ListSequence: Делегирован на уровень ниже - напрямую в LinkedList. Итератор хранит указатель на текущий узел (Node*), что позволяет переходить к следующему элементу за O(1)

5. Оператор []

Перегружен в базовом классе Sequence только для чтения (const T&). Это критически важно для Immutable-последовательностей, чтобы исключить случайное изменение данных через индекс
6. Свободные функции трансформации (utils.hpp)

- Операции split, zip и unzip вынесены из классов-контейнеров в свободные функции
- Данные операции изменяют базовый тип контейнера (например, Sequence<T> превращается в Sequence<Sequence<T>> при разбиении, или в Sequence<Pair<T, U>> при объединении)
